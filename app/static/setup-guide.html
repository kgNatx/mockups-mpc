Setup Guide

Mockups MPC

A self-hosted MCP server and gallery for design mockups. Connect your AI tools, send mockups via tool calls, and browse them in a beautiful gallery.

01

Quick Start

Mockups MPC is an MCP (Model Context Protocol) server that gives AI assistants the ability to store and manage design mockups. When connected, your AI tool gets six tools for sending, listing, updating, and managing mockups in a persistent gallery.

You need two things to get started:

  1. A running instance of the Mockups MPC server
  2. An MCP-compatible AI tool (Claude Code, Claude Desktop, etc.) configured to connect to it
02

Deploy the Server

Clone the repository and start the server with Docker Compose.

  1. Clone the repo and configure your domain:
    bash 
    git clone https://github.com/yourusername/mockups-mpc.git
cd mockups-mpc
cp .env.example .env
  2. Edit .env and set your domain:
    env 
    BASE_URL=https://mockups.yourdomain.com
  3. Start the server:
    bash 
    docker compose up -d
  4. Verify the server is running:
    bash 
    curl https://mockups.yourdomain.com/health
# {"status":"ok"}
03

Connect Claude Code

There are three ways to connect Claude Code to your Mockups MPC server. Pick whichever suits your workflow.

Easiest Method A: CLI Command

Add the server directly from your terminal. This is the fastest way to get connected.

bash 
claude mcp add-json mockups '{"type":"http","url":"https://mockups.yourdomain.com/mcp"}'

Or if you prefer SSE transport:

bash 
claude mcp add-json mockups '{"type":"sse","url":"https://mockups.yourdomain.com/mcp/sse"}'

Project Method B: Project-Level Config

Create a .mcp.json file in your project root. The server will only be available when working in that project.

.mcp.json 
{
  "mcpServers": {
    "mockups": {
      "type": "http",
      "url": "https://mockups.yourdomain.com/mcp"
    }
  }
}

Global Method C: Global Config

Add it to ~/.claude/.mcp.json to make the server available in all your projects.

~/.claude/.mcp.json 
{
  "mcpServers": {
    "mockups": {
      "type": "http",
      "url": "https://mockups.yourdomain.com/mcp"
    }
  }
}
Tip: Environment variable expansion
You can use environment variables in the URL to avoid hardcoding your domain:
~/.claude/.mcp.json 
{
  "mcpServers": {
    "mockups": {
      "type": "http",
      "url": "${MOCKUPS_URL:-https://mockups.yourdomain.com}/mcp"
    }
  }
}
04

Verify Connection

Once configured, verify that Claude Code can see the server.

  1. Start a Claude Code session in your terminal.
  2. Ask Claude to list your mockups:
    prompt 
    List all my mockups in the gallery.
  3. Claude should call the list_mockups tool and return results from your gallery. On a fresh install, you will see this Setup Guide as the only entry.
Expected behavior
When connected, Claude Code has access to 6 tools provided by the Mockups MPC server. You can verify by checking your MCP server list or asking Claude what tools are available from the mockups server.
05

Available Tools

The MCP server exposes six tools for managing your mockup gallery.

Tool Description Parameters
send_mockup Send a mockup to the gallery for permanent storage. Supports HTML, SVG, PNG, and JPG. project title content content_type description tags
list_mockups List mockups in reverse-chronological order, optionally filtered by project. project limit offset
get_mockup Get a specific mockup by its UUID, including view and gallery URLs. id
update_mockup Update a mockup's metadata or replace its content entirely. id title description tags content content_type
delete_mockup Delete a mockup by ID. Removes both the database record and the stored file. id
tag_mockup Add or remove tags on an existing mockup without replacing all tags. id add remove
Tip: Content types
For send_mockup, use html or svg with raw string content, and png or jpg with base64-encoded content.
06

Connect Claude Desktop

Claude Desktop uses a JSON config file for MCP servers. The file location depends on your OS:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json
  • Linux: ~/.config/Claude/claude_desktop_config.json

Open the file (create it if it doesn't exist) and add the mockups server:

claude_desktop_config.json 
{
  "mcpServers": {
    "mockups": {
      "type": "sse",
      "url": "https://mockups.yourdomain.com/mcp/sse"
    }
  }
}
Note: Transport type
Claude Desktop supports SSE transport for remote MCP servers. Use the /mcp/sse endpoint. If your version of Claude Desktop supports streamable HTTP, you can use "type": "http" with the /mcp endpoint instead.

After saving the config, restart Claude Desktop. The mockups tools will appear in the tools menu (hammer icon) when starting a new conversation.

07

Other AI Tools

Mockups MPC implements the standard Model Context Protocol, so any MCP-compatible client can connect. The server supports both Streamable HTTP and SSE transports.

  • Gemini CLI — configure MCP servers in your settings.json with the same URL pattern.
  • Any MCP client — point it to https://mockups.yourdomain.com/mcp using HTTP transport, or /mcp/sse for SSE.

The configuration shape is the same across all clients — you just need the URL and transport type.
